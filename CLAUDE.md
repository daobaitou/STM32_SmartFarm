# Project: 基于STM32的智能农业监测与灌溉控制系统

## 项目概述
- **类型**: 毕业设计项目
- **目标**: 基于STM32微控制器 + FreeRTOS，实现农业环境的智能监测与自动灌溉
- **技术栈**: C语言、STM32 HAL库、FreeRTOS、MQTT协议、QT框架
- **架构**: 端-边-云三层架构（嵌入式终端 + 腾讯云服务器 + QT/Web上位机）

---

## 硬件平台

### 主控芯片
| 参数 | 规格 |
|------|------|
| MCU | STM32F103C8T6 |
| Flash | 64KB |
| RAM | 20KB |
| 价格 | ~10元 |

### 传感器配置（核心功能）
| 传感器 | 型号 | 接口 | 精度/说明 |
|--------|------|------|----------|
| 空气温湿度 | DHT22 (AM2302) | 单总线 | ±0.5°C / ±2%RH |
| 土壤湿度 | FC28 | ADC | 模拟量输出 |
| 土壤温度 | DS18B20防水 | 单总线 | ±0.5°C |
| 光照强度 | BH1750 | I2C | 数字输出 |
| CO2浓度 | MH-Z19B | UART | ±50ppm |
| 大气压强 | BMP180 | I2C | 300-1100hPa |
| 雨滴检测 | LM393 | ADC/DO | 检测是否下雨 |

### 灌溉系统
| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 水流传感器 | YF-S201 | 脉冲计数 | 测量灌溉用水量 |
| 水位传感器 | 水滴检测板 | ADC | 监测水箱水位 |
| 继电器模块 | 5V光耦隔离 | GPIO | 控制水泵 |
| 水泵 | 12V直流 | 继电器驱动 | 灌溉执行 |

### 人机交互
| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 显示屏 | 0.96" OLED (SSD1306) | I2C | 128x64分辨率 |
| RTC时钟 | DS3231 | I2C | 精确时间戳 |
| 报警 | 蜂鸣器 + LED | GPIO | 声光报警 |
| 按键 | 4个轻触按键 | GPIO | 本地控制 |

### 通信模块
| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| WiFi | ESP8266 (ESP-01S) | UART | MQTT通信 |

### 电源系统（双电源架构）
- **主电源**: 12V适配器（市电）
- **备用电源**: 18650锂电池 × 2（7.4V）
- **切换机制**: 二极管OR电路，市电优先
- **断电检测**: GPIO中断（PA0）
- **断电响应**: 切换低功耗模式，降低采集频率

## FreeRTOS任务架构

### 任务优先级（高→低）
```
优先级6 | Task_Alarm        | 报警判断 + 断电响应
优先级5 | Task_MQTT_Sub     | MQTT订阅 + 命令接收
优先级4 | Task_Irrigation   | 灌溉逻辑 + 水流统计
优先级3 | Task_WaterLevel   | 水位监测 + 低水位报警
优先级2 | Task_Sensor       | 环境传感器采集
优先级2 | Task_Rain         | 雨滴检测 + 中断处理
优先级2 | Task_LCD          | OLED显示
优先级2 | Task_Key          | 按键扫描
优先级1 | Task_MQTT_Pub     | MQTT数据上传
```

### 消息队列
```
sensor_data_queue  → Task_Sensor → Task_LCD + Task_MQTT_Pub
control_cmd_queue  → Task_MQTT_Sub → Task_Irrigation
alarm_data_queue   → Task_Sensor/Task_WaterLevel → Task_Alarm
rain_event_queue   → Task_Rain → Task_Irrigation
```

### 采集周期
| 传感器 | 周期 | 备注 |
|--------|------|------|
| DHT22 | 2秒 | 空气温湿度 |
| DS18B20 | 2秒 | 土壤温度 |
| FC28 | 1秒 | 土壤湿度ADC |
| BH1750 | 1秒 | 光照强度 |
| MH-Z19B | 5秒 | CO2浓度UART |
| BMP180 | 2秒 | 大气压强 |
| YF-S201 | 实时 | 中断计数 |
| 水位 | 1秒 | ADC |
| 雨滴 | 实时 | 中断 |

## MQTT通信协议

### Topic设计
```
上行: smartfarm/{device_id}/sensor/data
      smartfarm/{device_id}/status/report
下行: smartfarm/{device_id}/control/mode
      smartfarm/{device_id}/control/pump
      smartfarm/{device_id}/config/threshold
```

### JSON数据格式
```json
// 传感器数据
{
  "device_id": "farm_001",
  "timestamp": 1711234567,
  "data": {
    "temperature": 25.6,
    "humidity": 65.2,
    "soil_moisture": 42,
    "soil_temperature": 22.1,
    "light": 850,
    "co2": 420,
    "pressure": 1013.2,
    "raindrop": 0,
    "water_flow": 0.5,
    "water_level": 75
  }
}
```

### QoS策略
- 传感器数据: QoS 0（允许丢失）
- 控制命令: QoS 1（至少一次）

## 开发约定

### 代码规范
- 使用C语言，遵循CMSIS和HAL库规范
- 函数命名：`模块_动作`（如 `Sensor_ReadTemperature`）
- 变量命名：驼峰式（项目内统一）
- 宏定义全部大写（如 `#define MAX_RETRY 3`）
- 每个模块包含头文件和源文件，头文件使用 `#ifndef` 保护

### 注释规范
- 文件头注释：作者、日期、模块功能描述
- 函数注释：功能说明、参数说明、返回值说明
- 关键算法：行内注释解释思路

### FreeRTOS 规范
- 任务命名：`vTask_模块名`
- 信号量命名：`xSemaphore_用途`
- 互斥量命名：`xMutex_资源`
- 队列命名：`xQueue_方向`

### 文件头模板
```c
/**
 * @file    sensor_dht22.c
 * @author  王国维
 * @date    2024-03-XX
 * @brief   DHT22温湿度传感器驱动模块
 * @note    使用单总线协议，需严格遵循时序要求
 */
```
