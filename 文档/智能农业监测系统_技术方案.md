> 文档生成时间：2024年3月
> 适用对象：物联网工程专业本科毕业设计

## 一、硬件方案设计

### 1.1 主控芯片

| 方案 | 说明 |
|------|------|
| **STM32F103C8T6**（推荐） | 成本低（~10元），资源够用，资料最多，适合毕设 |

> F103C8T6：64KB Flash + 20KB RAM，跑 FreeRTOS + MQTT 完全没问题。

### 1.2 环境监测传感器

| 传感器 | 型号 | 接口 | 说明 |
|--------|------|------|------|
| 空气温湿度 | **DHT22（AM2302）** | 单总线 | 精度 ±0.5°C / ±2%RH |
| 土壤湿度 | **FC28** 或电容式 | ADC | 模拟量输出 |
| 土壤温度 | **DS18B20防水** | 单总线 | 防水探头，精度±0.5°C |
| 光照强度 | **BH1750** | I2C | 数字输出，稳定可靠 |
| CO2浓度 | **MH-Z19B** | UART | 串口输出，精度±50ppm |
| 大气压强 | **BMP180** | I2C | 气压+海拔，300-1100hPa |
| 雨滴检测 | **LM393雨滴模块** | ADC/DO | 检测是否下雨 |

### 1.3 灌溉系统模块

| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 水流传感器 | **YF-S201** | 脉冲计数 | 测量灌溉用水量 |
| 水位传感器 | **水滴检测板** | ADC | 监测水箱水位 |
| 继电器模块 | **5V继电器+光耦** | GPIO | 控制水泵启停 |
| 水泵 | **12V直流水泵** | 继电器驱动 | 执行灌溉 |

### 1.4 人机交互模块

| 模块 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 显示模块 | **0.96寸OLED（SSD1306）** | I2C | 128x64分辨率 |
| 报警模块 | **蜂鸣器 + LED** | GPIO | 声光报警 |
| RTC时钟 | **DS3231** | I2C | 精确时间戳，断电保持 |

### 1.5 通信模块

### 1.6 电源系统设计（市电+电池备用）

本系统采用**双电源供电架构**，以市电为主电源，锂电池为备用电源，实现断电自动切换和低功耗备用模式。

#### 1.6.2 电源切换逻辑

| 状态 | 供电来源 | 系统模式 | 采集周期 | OLED | MQTT心跳 |
|------|----------|----------|----------|------|----------|
| **正常** | 市电 | 全功能 | 2秒 | 常亮 | 10秒 |
| **断电** | 电池 | 备用低功耗 | 30秒 | 关闭 | 60秒 |
| **恢复** | 市电 | 全功能 | 2秒 | 恢复 | 10秒 |

#### 1.6.3 断电检测与响应

```
断电检测电路：
  12V市电 ──┬──► LM2596 ──► 5V系统
            │
            └──► R1(10k) ──┬──► STM32 GPIO (PA0)
                           │
                           R2(3.3k)
                           │
                           GND

  当12V消失时，GPIO电平从高变低，触发外部中断
```

**软件响应流程**：
```
GPIO中断触发
    ↓
判断电平变化
    ├─ 低电平（断电）→ 进入备用模式
    │                  - 采集周期改为30秒
    │                  - 关闭OLED显示
    │                  - MQTT心跳延长到60秒
    │                  - 发送断电告警到云端
    │
    └─ 高电平（恢复）→ 恢复正常模式
                       - 采集周期恢复2秒
                       - 打开OLED显示
                       - MQTT心跳恢复10秒
                       - 发送恢复通知
```

#### 1.6.4 水泵供电说明

- 水泵使用12V独立供电（直接取自市电适配器）
- 断电时水泵无法工作（符合安全逻辑：断电不灌溉）
- 继电器控制信号来自STM32（3.3V逻辑电平）

## 二、软件架构设计（FreeRTOS）

### 2.1 任务划分

```
优先级  | 任务名称           | 堆栈 | 功能
--------|-------------------|------|------------------
6(最高) | Task_Alarm        | 256B | 报警判断 + 声光控制 + 断电检测响应
5       | Task_MQTT_Sub     | 512B | MQTT订阅 + 接收命令
4       | Task_Irrigation   | 512B | 灌溉逻辑 + 水流统计
3       | Task_WaterLevel   | 256B | 水位监测 + 低水位报警
2       | Task_Sensor       | 768B | 环境传感器采集(7个)
2       | Task_Rain         | 256B | 雨滴检测 + 中断处理
2       | Task_LCD          | 512B | OLED数据显示
2       | Task_Key          | 256B | 按键扫描 + 本地控制
1(最低) | Task_MQTT_Pub     | 512B | MQTT数据上传
```

### 2.2 消息队列设计

```
sensor_data_queue:  Task_Sensor → Task_LCD + Task_MQTT_Pub
control_cmd_queue:  Task_MQTT_Sub → Task_Irrigation
alarm_data_queue:   Task_Sensor/Task_WaterLevel → Task_Alarm
rain_event_queue:   Task_Rain → Task_Irrigation (下雨暂停灌溉)
```

### 2.3 采集策略

| 传感器 | 采集周期 | 说明 |
|--------|----------|------|
| DHT22 | 2秒 | 空气温湿度 |
| DS18B20 | 2秒 | 土壤温度 |
| FC28 | 1秒 | 土壤湿度(ADC) |
| BH1750 | 1秒 | 光照强度 |
| MH-Z19B | 5秒 | CO2浓度(UART) |
| BMP180 | 2秒 | 大气压强 |
| YF-S201 | 实时 | 水流脉冲计数(中断) |
| 水位 | 1秒 | ADC读取 |
| 雨滴 | 实时 | 中断触发 |

## 三、通信协议设计

### 3.1 MQTT Topic

```
# 上行
smartfarm/{device_id}/sensor/data      # 传感器数据
smartfarm/{device_id}/status/report    # 设备状态

# 下行
smartfarm/{device_id}/control/mode     # 模式切换
smartfarm/{device_id}/control/pump     # 水泵控制
smartfarm/{device_id}/config/threshold # 阈值设置
```

### 3.3 QoS选择

- 传感器数据：QoS 0
- 控制命令：QoS 1

### 3.4 云服务器部署方案

本系统采用**自有的腾讯云服务器**部署MQTT Broker和Web服务端，实现私有化部署，数据安全可控。

**服务器配置要求**：
- 操作系统：Ubuntu 20.04 / 22.04 LTS
- CPU：1核及以上
- 内存：1GB及以上
- 存储：20GB及以上
- 网络：公网IP，开放1883、5000端口

**部署服务清单**：

| 服务 | 软件 | 端口 | 说明 |
|------|------|------|------|
| MQTT Broker | **Mosquitto** | 1883 | 轻量级消息代理，处理设备通信 |
| Web服务端 | **Flask + Gunicorn** | 5000 | 提供API和Web页面 |
| 数据库 | **SQLite** | - | 文件型数据库，无需额外部署 |
| 进程管理 | **systemd** | - | 开机自启和进程守护 |

**私有化部署优势**：
1. 数据存储在自有服务器，安全性更高
2. 无依赖第三方服务，系统稳定性可控
3. 可根据需求扩展功能（如增加用户认证、数据备份等）
4. 答辩时体现完整的云端部署能力

### 4.2 SQLite数据表

```sql
CREATE TABLE sensor_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    temperature REAL,        -- 空气温度(°C)
    humidity REAL,           -- 空气湿度(%)
    soil_moisture REAL,      -- 土壤湿度(%)
    soil_temperature REAL,   -- 土壤温度(°C)
    light REAL,              -- 光照强度(lux)
    co2 REAL,                -- CO2浓度(ppm)
    pressure REAL,           -- 大气压强(hPa)
    raindrop INTEGER,        -- 雨滴状态(0/1)
    water_flow REAL,         -- 水流量(L)
    water_level INTEGER      -- 水位(%)
);

CREATE TABLE irrigation_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    mode TEXT,               -- 'auto' or 'manual'
    action TEXT,             -- 'on' or 'off'
    duration INTEGER,        -- 灌溉时长(秒)
    water_used REAL,         -- 用水量(升)
    soil_before REAL,        -- 灌溉前土壤湿度
    soil_after REAL          -- 灌溉后土壤湿度
);
```

**部署步骤**：

```bash
# 1. 连接服务器
ssh root@你的服务器公网IP

# 2. 安装环境
apt update && apt install -y python3 python3-pip mosquitto
pip3 install flask paho-mqtt gunicorn

# 3. 启动Mosquitto
systemctl start mosquitto
systemctl enable mosquitto

# 4. 部署Flask应用（详见附录）
# 5. 配置防火墙开放1883、5000端口
# 6. 配置systemd实现开机自启
```

**访问方式**：
- Web页面：`http://你的服务器IP:5000`

## 七、开题报告内容

### 1、毕业设计（论文）的主要内容

本课题设计并实现一套基于STM32的智能农业监测与灌溉控制系统，涵盖嵌入式终端、云端通信和远程上位机三个层面。主要研究内容如下：

**（1）硬件系统设计**

以STM32F103C8T6微控制器为核心，设计多传感器环境参数采集模块，包括DHT22空气温湿度传感器、FC28土壤湿度传感器、DS18B20土壤温度传感器、BH1750光照强度传感器、MH-Z19B二氧化碳浓度传感器、BMP180大气压强传感器和LM393雨滴检测模块。灌溉系统方面采用YF-S201水流传感器测量灌溉用水量，配合水位传感器监测水箱水位防止干烧。通信方面采用ESP8266 WiFi模块通过UART与主控通信，接入MQTT协议实现设备上云。此外设计了继电器驱动模块控制水泵执行灌溉动作，OLED显示模块和蜂鸣器报警模块用于现场人机交互，DS3231 RTC模块提供精确时间戳，轻触按键支持本地手动控制。**电源系统采用市电+电池双电源架构，实现断电自动切换和低功耗备用模式，保障系统供电可靠性。**

**（2）嵌入式软件设计**

基于FreeRTOS嵌入式实时操作系统进行多任务软件开发，设计传感器采集、MQTT通信、灌溉控制、LCD显示、报警判断和电源管理等多个并发任务，通过消息队列和信号量实现任务间数据共享与同步。**设计双电源供电管理机制，通过GPIO中断检测市电状态，断电时自动切换到电池备用并进入低功耗模式，降低采集频率、关闭非必要外设以延长续航时间。**同时设计断网数据缓存机制和断电参数恢复功能，保证系统在通信中断和供电异常情况下的可靠性。

**（3）远程监控平台设计**

开发基于QT框架的桌面端上位机软件，实现传感器数据的实时展示、多参数历史曲线绘制、自动/手动灌溉模式切换和报警阈值配置等功能。同时基于Flask框架开发轻量级Web服务端，支持通过手机浏览器远程访问系统数据和发送控制指令。

**（4）智能决策功能**

集成第三方天气数据服务接口，根据天气预报信息（降雨概率、温度趋势）智能调整灌溉策略，实现基于环境综合信息的决策型灌溉控制，提升系统的智能化水平。

| **上行数据** | 传感器 → FreeRTOS队列 → MQTT发布 → 云端 → 上位机显示 | 环境数据实时上报 |
| **下行控制** | 上位机 → 云端MQTT → STM32订阅 → 执行器动作 | 远程灌溉控制 |
| **智能决策** | 天气API → 降雨分析 → 策略下发 → 本地执行 | 智能灌溉调度 |
| **断电响应** | 市电检测 → GPIO中断 → 切换备用模式 | 供电连续性保障 |

感知层以STM32F103C8T6为主控，在FreeRTOS操作系统上运行传感器采集、灌溉控制和本地显示等任务，通过ESP8266模块连接WiFi网络，以MQTT协议将传感器数据和设备状态上传至腾讯云服务器上的Mosquitto消息代理，同时接收来自上位机的控制命令。

应用层分为两个部分：QT桌面端提供完整的监控管理界面，包括实时数据展示、历史曲线、灌溉控制和数据导出等功能；Flask Web端部署在腾讯云服务器上，提供移动端访问入口，通过REST API和MQTT桥接实现与设备端的间接通信，支持手机浏览器远程访问。

数据存储采用双端方案：STM32端利用内部Flash存储系统配置和断网缓存数据，保障数据安全；腾讯云服务器端采用SQLite数据库存储历史传感器数据和灌溉日志，支持数据查询、图表展示和报表导出。

**预期目标：**

1. 实现多传感器环境参数（温度、湿度、土壤温湿度、光照强度、CO2浓度、大气压强、降雨检测）的实时采集与本地OLED显示
2. 实现灌溉用水量统计和水箱水位监测，低水位自动报警，防止干烧
3. 基于MQTT协议实现嵌入式终端与远程上位机的双向通信，通过腾讯云服务器部署私有化MQTT Broker，通信延迟不超过2秒
4. 实现自动灌溉和手动灌溉两种控制模式，支持降雨检测自动暂停灌溉，土壤湿度达标自动关闭水泵
5. QT上位机实现实时监控、多参数历史曲线、灌溉控制、数据导出和报表生成等完整功能
6. 开发Web端页面部署至腾讯云服务器，支持手机浏览器远程访问
7. 集成天气数据服务，结合本地雨滴检测实现双重降雨判断，智能调整灌溉策略
8. 系统具备双电源供电能力，市电优先、电池备用，断电自动切换到低功耗模式，保障系统连续运行

### 5、完成课题所需条件及落实措施

**（1）硬件条件**

本系统所需硬件主要包括：STM32F103C8T6最小系统板、ESP8266 WiFi模块、DHT22空气温湿度传感器、FC28土壤湿度传感器、DS18B20土壤温度传感器、BH1750光照传感器、MH-Z19B二氧化碳传感器、BMP180大气压强传感器、LM393雨滴检测模块、YF-S201水流传感器、水位传感器、0.96寸OLED显示屏、DS3231 RTC时钟模块、5V继电器模块、12V直流水泵、蜂鸣器及LED报警模块、轻触按键、杜邦线和面包板等。**电源系统采用12V适配器（市电）+18650锂电池（备用）双电源架构，配合LM2596降压模块和MT3608升压模块实现自动切换。**上述硬件均可通过淘宝等电商平台采购，总成本约为270元。此外需要一台个人计算机用于软件开发和程序烧录，以及自有的腾讯云服务器用于部署MQTT Broker和Web服务端。

**（2）软件条件**

**（3）网络与平台条件**

系统采用**自有的腾讯云服务器**作为云端平台，实现私有化部署。服务器配置为Ubuntu操作系统，部署Mosquitto作为MQTT消息代理服务器，部署Flask作为Web服务端，数据库采用SQLite。相比使用公共MQTT服务器，私有化部署具有以下优势：数据存储在自有服务器，安全性更高；无依赖第三方服务，系统稳定性可控；可根据需求灵活扩展功能。天气数据接口使用和风天气提供的免费API（每日1000次调用额度），满足系统智能决策需求。

### 8.3 Flask应用部署

**项目结构**：
```
/opt/smartfarm/
├── app.py              # Flask主程序
├── templates/
│   └── index.html      # Web页面
├── farm_data.db        # SQLite数据库（自动生成）
└── venv/               # Python虚拟环境
```

**app.py 核心代码**：
```python
from flask import Flask, render_template, jsonify, request
import sqlite3
import paho.mqtt.client as mqtt
import json
import threading
from datetime import datetime

app = Flask(__name__)
MQTT_BROKER = "localhost"
MQTT_PORT = 1883

def init_db():
    conn = sqlite3.connect('/opt/smartfarm/farm_data.db')
    conn.execute('''CREATE TABLE IF NOT EXISTS sensor_data (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
        temperature REAL, humidity REAL, soil_moisture REAL,
        light REAL, co2 REAL
    )''')
    conn.commit()
    conn.close()

def on_message(client, userdata, msg):
    try:
        data = json.loads(msg.payload)
        conn = sqlite3.connect('/opt/smartfarm/farm_data.db')
        conn.execute('''INSERT INTO sensor_data
            (temperature, humidity, soil_moisture, light, co2)
            VALUES (?,?,?,?,?)''',
            (data['data']['temperature'], data['data']['humidity'],
             data['data']['soil_moisture'], data['data']['light'],
             data['data']['co2']))
        conn.commit()
        conn.close()
    except Exception as e:
        print(f"Error: {e}")

# MQTT客户端线程
mqtt_client = mqtt.Client()
mqtt_client.on_message = on_message
mqtt_client.connect(MQTT_BROKER, MQTT_PORT, 60)
mqtt_client.subscribe("smartfarm/+/sensor/data")
mqtt_thread = threading.Thread(target=mqtt_client.loop_forever)
mqtt_thread.daemon = True
mqtt_thread.start()

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/api/latest')
def get_latest():
    conn = sqlite3.connect('/opt/smartfarm/farm_data.db')
    cursor = conn.execute("SELECT * FROM sensor_data ORDER BY timestamp DESC LIMIT 1")
    row = cursor.fetchone()
    conn.close()
    if row:
        return jsonify({
            "temperature": row[2], "humidity": row[3],
            "soil_moisture": row[4], "light": row[5],
            "co2": row[6], "timestamp": row[1]
        })
    return jsonify({"error": "no data"})

@app.route('/api/pump', methods=['POST'])
def control_pump():
    action = request.json.get('action')
    mqtt_client.publish("smartfarm/farm_001/control/pump",
                        json.dumps({"action": action, "timestamp": int(datetime.now().timestamp())}))
    return jsonify({"status": "ok", "action": action})

if __name__ == '__main__':
    init_db()
    app.run(host='0.0.0.0', port=5000, debug=False)
```

### 8.4 systemd服务配置（开机自启）

[Unit]

After=network.target mosquitto.service

[Service]
Type=simple
WorkingDirectory=/opt/smartfarm
ExecStart=/usr/bin/python3 /opt/smartfarm/app.py
Restart=always
RestartSec=5
User=root

[Install]
WantedBy=multi-user.target
EOF
