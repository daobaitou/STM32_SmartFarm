# 基于STM32的智能农业监测与灌溉控制系统 — 详细开发计划书

> 编写时间：2026年4月17日
> 适用阶段：毕业设计实施阶段

---

## 一、开发策略原则

### 1.1 核心原则

1. **分阶段、模块化开发**：不追求一次完成所有配置，每个阶段独立验证后再进入下一阶段
2. **先验证后集成**：每个模块单独测试通过后再集成到主工程
3. **从简单到复杂**：从LED闪烁开始，逐步增加功能
4. **保留里程碑版本**：每个阶段完成后保存独立工程备份

### 1.2 开发顺序

```
阶段0: 工程框架搭建 → LED闪烁验证
阶段1: 外设驱动开发（无RTOS，裸机轮询）
阶段2: FreeRTOS集成与任务划分
阶段3: 传感器任务集成
阶段4: 灌溉控制与通信任务
阶段5: 系统联调与优化
阶段6: 上位机与云端开发（并行）
```

```
模块分类        引脚           功能                配置说明
─────────────────────────────────────────────────────────────

SYS            PA13/PA14      SWD调试            Serial Wire

定时器         PA0            TIM2_CH1_ETR       YF-S201水流脉冲

GPIO输出       PC13           GPIO_Output        板载LED

               PB8            GPIO_Output        蜂鸣器
               PB9            GPIO_Output        LED红灯

GPIO输入       PB12           GPIO_Input (PU)    KEY1
               PB13           GPIO_Input (PU)    KEY2
               PB14           GPIO_Input (PU)    KEY3
               PB15           GPIO_Input (PU)    KEY4

外部中断       PA2            GPIO_EXTI2         断电检测
```

**时钟配置：**
```
HSE: 8MHz (外部晶振)
PLL: x9 倍频
SYSCLK: 72MHz
APB1: 36MHz (定时器2x = 72MHz)
APB2: 72MHz
```

**需要开启的中断：**
- EXTI line[2:1] (PA2断电检测)
- TIM2 global interrupt (水流脉冲)
- USART1 global interrupt (ESP8266)
- USART3 global interrupt (MH-Z19B)

### 2.4 需避免的引脚（Blue Pill JTAG冲突）

```
禁止使用: PA15, PB3, PB4 (JTAG引脚，即使SWD模式也有冲突)
谨慎使用: PA8 (可能有时钟相关冲突)
```

### 3.2 Task_Sensor任务实现

```c
void vTask_Sensor(void *pvParameters)
{
    SensorData_t sensorData;
    TickType_t xLastWakeTime = xTaskGetTickCount();

    while (1) {
        // 采集所有传感器数据
        DHT22_Read(&dht22Data);
        DS18B20_Read(&ds18b20Data);
        BH1750_Read(&light);
        BMP180_Read(&bmp180Data);
        MHZ19B_Read(&co2Data);
        FC28_Read(&soilMoisture);
        ReadWaterLevel(&waterLevel);
        ReadRainStatus(&rainStatus);

        // 填充数据结构
        sensorData.temperature = dht22Data.temperature;
        sensorData.humidity = dht22Data.humidity;
        // ... 其他字段

        // 发送到队列
        xQueueOverwrite(xQueue_SensorData, &sensorData);

        // 固定周期2秒
        vTaskDelayUntil(&xLastWakeTime, pdMS_TO_TICKS(2000));
    }
}
```

## 十一、风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 传感器驱动时序问题 | 中 | 高 | 先用逻辑分析仪验证时序 |
| FreeRTOS堆栈溢出 | 中 | 高 | 使用`uxTaskGetStackHighWaterMark()`监控 |
| ESP8266通信不稳定 | 中 | 高 | 增加重试机制和看门狗 |
| 内存不足 | 低 | 高 | 优化任务堆栈，使用静态分配 |
| 云服务器部署问题 | 低 | 中 | 先本地测试，再部署云端 |
