> 编写时间：2026年4月17日
> 适用阶段：毕业设计实施阶段

---

## 一、开发策略原则

### 1.1 核心原则

1. **分阶段、模块化开发**：不追求一次完成所有配置，每个阶段独立验证后再进入下一阶段
2. **先验证后集成**：每个模块单独测试通过后再集成到主工程
3. **从简单到复杂**：从LED闪烁开始，逐步增加功能
4. **保留里程碑版本**：每个阶段完成后保存独立工程备份

### 1.2 开发顺序

```
阶段0: 工程框架搭建 → LED闪烁验证
阶段1: 外设驱动开发（无RTOS，裸机轮询）
阶段2: FreeRTOS集成与任务划分
阶段3: 传感器任务集成
阶段4: 灌溉控制与通信任务
阶段5: 系统联调与优化
阶段6: 上位机与云端开发（并行）
```

**实际任务分配：**
```c
#define PRIORITY_SENSOR     2   // 传感器采集
#define PRIORITY_LCD        2   // OLED显示

#define PRIORITY_LED        0   // LED闪烁
```

**任务栈与功能：**
| 任务 | 栈(字) | 优先级 | 功能 |
|------|--------|--------|------|
| vTask_Sensor | 512 | 2 | 每2秒采集所有传感器，发送队列 |
| vTask_LCD | 384 | 2 | 接收数据，OLED显示（I2C互斥保护） |
| vTask_Print | 256 | 1 | 接收数据，串口打印 |
| vTask_LED | 128 | 0 | PC13每500ms翻转 |

**数据通信：**
- 队列：`xQueue_SensorData`（长度1，覆盖式写入）
- 互斥量：`xMutex_I2C`（保护共享I2C总线）

**SensorData_t 结构：**
```c
typedef struct {
    float temperature, humidity, soil_temp, pressure, bmp_temp;
    float light, flow_rate, total_volume;
    uint8_t soil_moisture;
    uint16_t adc_raw;
    uint32_t timestamp;
} SensorData_t;
```

#### 2.3 关键技术要点

1. **SysTick冲突解决**：重写`HAL_InitTick()`用TIM4替代SysTick
2. **中断向量映射**：SysTick_Handler→xPortSysTickHandler, PendSV→xPortPendSVHandler, SVC→vPortSVCHandler
3. **TIM4配置**：72分频→1MHz，周期1000→1ms，NVIC优先级最低(15)
4. **I2C互斥保护**：LCD任务通过mutex独占I2C总线，防止与BH1750/BMP180冲突
5. **Windows大小写问题**：`freertos.h`与`FreeRTOS.h`冲突，改名为`app_tasks.h`

```c
typedef enum {
    MODE_AUTO,      // 自动模式
    MODE_MANUAL     // 手动模式
} IrrigationMode_t;

void vTask_Irrigation(void *pvParameters)
{
    while (1) {
        if (mode == MODE_AUTO) {
            // 自动灌溉逻辑
            if (soilMoisture < threshold_low && !rain && waterLevel > 20) {
                Pump_On();
            } else if (soilMoisture > threshold_high) {
                Pump_Off();
            }
        }
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```
